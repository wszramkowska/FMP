# FMP API Documentation Pipeline

This folder contains the documentation source used by the automated Doxygen
pipeline. Documentation is generated on GitHub Actions, not on developer
machines.

## How Generation Works

Pull requests run `.github/workflows/docs-pr.yml`, which installs Doxygen and
Graphviz on the GitHub-hosted runner, generates the API site, and uploads the
HTML output as an artifact.

Pushes to `main` run `.github/workflows/docs.yml`, generate the same Doxygen
site, upload it as a GitHub Pages artifact, and deploy it with:

- `actions/configure-pages`
- `actions/upload-pages-artifact`
- `actions/deploy-pages`

The published site is available at:

```text
https://<org-or-user>.github.io/<repo>/
```

## How To Extend Documentation

- Document public Unreal C++ declarations directly in public headers.
- Keep comments beside `UCLASS`, `USTRUCT`, `UENUM`, `UINTERFACE`, `UFUNCTION`,
  `UPROPERTY`, and delegate declarations.
- Use concrete behavior-focused tags such as `@brief`, `@param`, `@return`,
  `@note`, `@warning`, `@see`, and `@ingroup`.
- Add module or system groups to `groups.dox` as the project grows.
- Add landing-page or architecture prose to `mainpage.dox`.
- Do not commit `docs/generated`; it is generated by CI.

## Quality Gates

The Doxygen configuration enables warning output for undocumented APIs,
incomplete parameter docs, and malformed references. Catastrophic generation
failures stop the workflow, and generated HTML must include an `index.html`
before artifacts or Pages deployment proceed.
