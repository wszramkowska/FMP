# FMP API Documentation

This page is the documentation hub for FMP. The full API reference is generated
with Doxygen by GitHub Actions and published to GitHub Pages.

## Open The Generated Documentation

- [Published API documentation](https://wszramkowska.github.io/FMP/)
- [Generated Doxygen index](generated/html/index.html)
- [Documentation source landing page](index.md)

If GitHub Pages is configured to publish from the repository `docs` folder
instead of GitHub Actions, use the generated Doxygen index link above. The
preferred Pages source is **GitHub Actions**, which publishes the generated
`docs/generated/html` artifact directly.

## Documentation Sources

- `mainpage.dox` defines the Doxygen front page.
- `groups.dox` defines API groups used by documented Unreal types.
- `index.md` is the repository-facing overview for documentation maintainers.
- `custom.css` and `DoxygenLayout.xml` adjust the generated site presentation.

## How Generation Works

Pull requests run `.github/workflows/docs-pr.yml`, install Doxygen and Graphviz,
generate the API site, and upload the HTML output as a workflow artifact.

Pushes to `main` run `.github/workflows/docs.yml`, generate the same Doxygen
site, upload `docs/generated/html` as a GitHub Pages artifact, and deploy it
through the `github-pages` environment.

## How To Extend Documentation

- Document public Unreal C++ declarations directly in public headers.
- Keep comments beside `UCLASS`, `USTRUCT`, `UENUM`, `UINTERFACE`, `UFUNCTION`,
  `UPROPERTY`, and delegate declarations.
- Use behavior-focused tags such as `@brief`, `@param`, `@return`, `@note`,
  `@warning`, `@see`, and `@ingroup`.
- Add module or system groups to `groups.dox` as the project grows.
- Add landing-page or architecture prose to `mainpage.dox`.
- Do not commit `docs/generated`; it is generated by CI.
